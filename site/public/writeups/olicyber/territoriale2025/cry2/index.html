Cry2 — Italian Wordlist Passphrase

The Challenge#

The challenge ships a challenge.py that generates passphrases by mapping each letter a–z to one of 26 Italian words. A passphrase is just those words joined by hyphens. You get a passphrase.txt full of hyphen-separated words, and you have to turn it back into text — which is the flag.

The word list is:

3

casa, albero, notte, sole, montagna, fiume, mare, vento, nuvola, pioggia,
strada, amico, sorriso, viaggio, tempo, cuore, stella, sogno, giorno,
libro, porta, luce, ombra, silenzio, fiore, luna

Index 0 → a, index 1 → b, …, index 25 → z. “casa” encodes a, “luna” encodes z.

Approach#

The challenge.py actually ships the reverse_passphrase function right alongside the encoder, so there isn’t much cryptanalysis needed. You build the inverse dict — word to its index — split the passphrase on -, map each word to chr(index + ord('a')), and concatenate. That’s the whole thing.

The encoding is a simple substitution cipher disguised as something exotic by using Italian words instead of letters. Since both the word list and the separator are fixed and known, reversing it is a one-liner. The only realistic failure mode is a typo in the passphrase or an unexpected word, but the challenge guarantees a clean input.

It’s worth noting that this scheme has zero security: the word list is given, there are only 26 possible symbols, and there’s no key. Anyone with the word list can decode any passphrase instantly.

Solution#

The challenge.py file already contains everything needed:

words = ["casa","albero","notte","sole","montagna","fiume","mare","vento","nuvola","pioggia",
         "strada","amico","sorriso","viaggio","tempo","cuore","stella","sogno","giorno",
         "libro","porta","luce","ombra","silenzio","fiore","luna"]

def reverse_passphrase(passphrase_str):
    words_dict = {w: i for i, w in enumerate(words)}
    return ''.join(chr(words_dict[w] + ord('a')) for w in passphrase_str.split('-') if w in words_dict)

with open("passphrase.txt") as rf:
    print(reverse_passphrase(rf.read()))

Running it prints the flag directly. The reverse_passphrase function constructs a {word: index} dict, splits the input on -, looks up each word, adds the ASCII offset of 'a' (97), and joins the characters. Any word not in the list is silently ignored thanks to the if w in words_dict guard, which handles any trailing newline in the file without special-casing it.

What I Learned#

When a challenge ships the encoder and the decoder in the same file, read the whole file before writing anything — the answer is already there. Wordlist-based encodings feel exotic but break down to a lookup table; the “crypto” label is often misleading.
