The Challenge#

The server hands you a hex-encoded ciphertext. The encryption pipeline is: take each plaintext byte, XOR it with a repeating key of unknown length, then put the result through a 256-entry nonlinear S-Box. You’re given the S-Box and the encrypted output, and you have to recover the plaintext — which contains the flag delimited by |.

Approach#

The structure here is a classic repeating-key XOR, just with an extra layer of indirection from the S-Box. The S-Box doesn’t prevent frequency analysis; it just means you have to peel it off first before you can reason about the underlying key bytes.

My plan was to brute-force key lengths from 6 to 12. For each candidate length L, I split the ciphertext into L independent streams: positions 0, L, 2L, … all share key byte 0; positions 1, L+1, 2L+1, … all share key byte 1; and so on. For each stream, I try all 256 possible key bytes: XOR every byte in the stream with the candidate, then apply the inverse S-Box. If the result looks like printable English text, the key byte is likely correct.

The scoring function gives +1 for any printable ASCII byte and an extra +1 for letters and spaces, then -5 for anything non-printable. That’s a heavy penalty for garbage bytes, which makes the signal clear enough even for short streams. After resolving the key, I apply a tiebreaker: if the decoded plaintext doesn’t contain |, I subtract 100 from its score. Flag format guarantees that delimiter is there, so this nudges the solver toward the correct key length without complicating the inner loop.

INV_SBOX = [0]*256
for i, val in enumerate(SBOX):
    INV_SBOX[val] = i

def xor_bytes(data, key):
    return bytes(b ^ key[i%len(key)] for i, b in enumerate(data))

def decrypt(ciphertext, key):
    xored = xor_bytes(ciphertext, key)
    return bytes(INV_SBOX[b] for b in xored)

def score_text(text):
    score = 0
    for b in text:
        if 32 <= b <= 126:
            score += 1
            if chr(b) in string.ascii_letters + " ":
                score += 1
        else:
            score -= 5
    return score

def break_repeating_xor(ciphertext):
    best_score, best_plaintext, best_key = float('-inf'), None, None
    for key_len in range(6, 13):
        candidate_key = bytearray(key_len)
        for i in range(key_len):
            block = ciphertext[i::key_len]
            best_ks, best_kb = float('-inf'), 0
            for candidate in range(256):
                trial = bytes(INV_SBOX[b ^ candidate] for b in block)
                s = score_text(trial)
                if s > best_ks:
                    best_ks, best_kb = s, candidate
            candidate_key[i] = best_kb
        plaintext = decrypt(ciphertext, candidate_key)
        s = score_text(plaintext)
        if b"|" not in plaintext:
            s -= 100
        if s > best_score:
            best_score, best_plaintext, best_key = s, plaintext, bytes(candidate_key)
    return best_key, best_plaintext

def main():
    with open("again_output.txt") as f:
        ciphertext = binascii.unhexlify(f.read().strip())
    print("[*] Attempting to break the encryption...")
    key, plaintext = break_repeating_xor(ciphertext)
    print(f"[*] Key (hex): {key.hex()}")
    print(f"[*] Plaintext: {plaintext}")
    if b"|" in plaintext:
        flag_part = plaintext.split(b"|", 1)[1].strip()
        print(f"\n[+] Flag: flag{{{flag_part.decode()}}}")

if __name__ == "__main__":
    main()

The inverse S-Box is precomputed at startup so the inner loop stays fast — 256 candidates × 12 key lengths × every position in the ciphertext is still well under a million operations. The key insight is that the S-Box is deterministic and invertible, so once you know the XOR key byte for a position, applying INV_SBOX recovers the plaintext byte exactly. The frequency scoring does the rest.

What I Learned#

Layering a substitution table over XOR doesn’t add meaningful security if the substitution is fixed and known — you just factor it out during analysis. The structure of the attack is identical to breaking raw repeating-key XOR; the S-Box only adds one extra lookup per byte.
