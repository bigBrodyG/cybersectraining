Blocky OX — CBC Decryption Oracle as Encryption Oracle

March 1, 2026 · 3 min · bigBrodyG

The Challenge#

chall.py exposes an AES-CBC decryption oracle. You send hex-encoded IV || ciphertext; the server decrypts it with a fixed secret key and returns ptx.hex(). If the decrypted plaintext equals pad(target, 16) — where target is the full text of the GNU/Linux manifesto, padded to a block boundary — the server prints the flag instead.

So the challenge is: construct a valid AES-CBC encryption of a known, long plaintext, without knowing the key.

Approach#

AES-CBC decryption works block by block. For block i, the plaintext is:

$$P_i = \text{BlockDecrypt}(K, C_i) \oplus C_{i-1}$$

(with $C_0 = IV$). Each block’s decryption depends only on $K$ and $C_i$; the XOR with the previous ciphertext block is applied afterward. This means if I query the oracle with an arbitrary ciphertext block C and an IV of all zeros, the oracle computes:

$$P = \text{BlockDecrypt}(K, C) \oplus 0^{16} = \text{BlockDecrypt}(K, C)$$

That’s a raw block decryption — and it gives me access to BlockDecrypt(K, ·) without knowing K.

To encrypt a target plaintext block $T_i$, I pick a random ciphertext block $C_i$ and query the oracle with IV=0 to get $D_i = \text{BlockDecrypt}(K, C_i)$. Then I set the “previous block” (i.e., IV for the first block, or C_{i-1} for subsequent blocks) to $D_i \oplus T_i$, so that:

$$P_i = D_i \oplus (D_i \oplus T_i) = T_i \checkmark$$

By chaining this process I build a full IV || C_1 || C_2 || ... that decrypts to the padded target. Each target block requires one oracle query.

Solution#

No standalone exploit script exists for this one — the attack was worked out conceptually and traced through the first few blocks manually during the competition.

# Pseudocode
target_blocks = split_into_16byte_blocks(pad(target, 16))
ciphertext_blocks = []
prev_block = None  # will become IV

for T_i in reversed(target_blocks):
    C_i = os.urandom(16)          # arbitrary ciphertext block
    D_i = oracle_decrypt(C_i)     # BlockDecrypt(K, C_i)
    prev_i = xor(D_i, T_i)       # the block that XORs to give T_i
    ciphertext_blocks.insert(0, C_i)
    prev_block = prev_i

iv = prev_block
send_to_server(iv + b''.join(ciphertext_blocks))

The oracle call per block is the bottleneck — for the full manifesto text this is on the order of dozens to a couple hundred queries, all feasible interactively.

What I Learned#

A decryption oracle is just as powerful as an encryption oracle when you understand what CBC decryption actually computes at the block level. The key insight — that BlockDecrypt(K, C) XOR 0 = BlockDecrypt(K, C) — turns the oracle into a free BlockDecrypt primitive, and from there constructing arbitrary encryptions is mechanical.
