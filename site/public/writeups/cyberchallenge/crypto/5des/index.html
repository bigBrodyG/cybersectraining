5DES — Controlled Key Derivation Attack

January 1, 2025 · 3 min · bigBrodyG
Table of Contents

The Challenge#

5des.py implements a custom 5-key DES cipher: a series of five DES operations alternating encrypt/decrypt with five independent 8-byte subkeys (40 bytes total). The server exposes three operations:

  1. Update the 5DES key via new_5des_key(user_plain, user_key) — computes DES(user_key).encrypt(DES(user_key).encrypt(user_plain)); the result becomes the new 40-byte 5DES key
  2. Encrypt the flag with the current 5DES key and save the ciphertext
  3. Decrypt the last saved ciphertext with the current 5DES key and print the result

To get the flag, I need option 5 to decrypt something that contains the flag. But option 5 decrypts the last saved ciphertext with the current key — which means if I encrypt the flag (option 3) and then change the key (option 1), option 5 will decrypt the flag ciphertext with the wrong key and produce garbage.

The attack path: set a known 5DES key first, then encrypt the flag, then decrypt it — all with the same known key. Or: encrypt the flag, record the ciphertext, change the key to a known value, then ask for a decrypt of something that produces the flag.

Approach#

The key insight is that new_5des_key(plain, key) gives me complete control over the new 40-byte 5DES key, because I choose both plain and key. Concretely:

$$\text{new_key} = \text{DES}(k).\text{encrypt}(\text{DES}(k).\text{encrypt}(p))$$

I choose p and k, so I can precompute this value offline. If I want the new 5DES key to be all zeros, I just solve: what p gives me \x00 * 8 after two DES encryptions with key k? Or more practically, I just pick arbitrary p and k, compute the resulting 40-byte key offline, and I know the key.

Then the sequence is:

  1. Call option 1 with known p and k; compute the 40-byte key offline
  2. Call option 3 to encrypt the flag under my known key; record the ciphertext
  3. Call option 1 again with the same p and k (same key)
  4. Call option 5 — it decrypts the saved ciphertext with my known key → flag

Since I know the 40-byte key, I can also just decrypt the ciphertext locally. But option 5 is simpler.

from Crypto.Cipher import DES
from pwn import *
import os

def new_5des_key_local(plain, key):
    """Compute the 40-byte key as the server would."""
    d = DES.new(key, DES.MODE_ECB)
    step1 = d.encrypt(plain)
    step2 = d.encrypt(step1)
    return step2 * 5   # server repeats the 8-byte block 5 times to fill 40 bytes

# Choose arbitrary plain and key (both 8 bytes)
p = b'\x41' * 8
k = b'\x42' * 8
derived_key = new_5des_key_local(p, k)

r = remote('5des.chall.cyberchallenge.it', PORT)

# Step 1: set key to our known value
r.sendlineafter(b'> ', b'1')
r.sendlineafter(b'plain: ', p.hex().encode())
r.sendlineafter(b'key: ', k.hex().encode())

# Step 3: encrypt flag (saves flag ciphertext)
r.sendlineafter(b'> ', b'3')

# Step 5: decrypt saved flag ciphertext → flag (key is still our known key)
r.sendlineafter(b'> ', b'5')
print(r.recvall().decode())

The server uses the 5DES key I set in step 1 for both option 3 and option 5, as long as I don’t call option 1 again in between. The flag comes back in the option 5 output.

What I Learned#

When an attacker controls a key derivation function’s inputs completely, they effectively control the key. “Controlled key derivation” isn’t key security — a key derivation function is only useful if the inputs include at least one secret that the attacker doesn’t know (like a password or master key). Here, both inputs are attacker-supplied.
