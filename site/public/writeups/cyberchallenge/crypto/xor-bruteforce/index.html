XOR Bruteforce

January 1, 2025 · 2 min · bigBrodyG
Table of Contents

The Challenge#

A hex-encoded ciphertext encrypted with a single-byte XOR key. All 256 possible key bytes need to be tried; one of them produces readable plaintext containing the flag.

Approach#

Single-byte XOR is the most basic cipher in the CTF toolkit. Every byte of the ciphertext is XOR’d with the same 1-byte key. Since there are only 256 possible keys, you don’t need frequency analysis or anything clever — you just try all 256 and look at which output is printable English.

The chepy library is a Python port of CyberChef with a chainable API. It has a dedicated xor_bruteforce() operation that iterates all 256 key bytes and returns the results, typically highlighting the one with the highest printable-ASCII ratio or letting you filter visually.

This is the kind of challenge where the “solution” is knowing what tool to reach for, not writing any algorithm.

Solution#

from chepy import Chepy

with open("ciphertext.txt") as f:
    hex_ct = f.read().strip()

c = Chepy(hex_ct).from_hex().xor_bruteforce()
print(c.o)

from_hex() converts the hex string to raw bytes. xor_bruteforce() tries all 256 keys and returns a list of (key, plaintext) pairs. The output with a key around printable ASCII range — or simply the one that looks like English text — contains the flag.

If the output is noisy, filter for lines containing flag{:

for key, plaintext in c.o:
    if b'flag{' in plaintext:
        print(f"Key: {key}, Flag: {plaintext}")
        break

Running this against the ciphertext yields the flag directly.

What I Learned#

chepy wraps tedious one-liners into a chainable interface. For single-byte XOR specifically, xor_bruteforce() is faster to type than writing the loop yourself — useful when you’re moving fast during a competition. Know your tools.
